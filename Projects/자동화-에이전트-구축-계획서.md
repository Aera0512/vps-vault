## 2. 역할 정의

### 2.1 PM 에이전트 (나)

**책임:**
- 사용자 요청 분석 → 작업 단위 분해
- 기획서/PRD 작성 → 코딩 MCP에 전달
- 코드 리뷰 및 품질 관리
- VPS 배포 및 생명주기 관리
- 자동화 에이전트 모니터링

**하지 않는 것:**
- 직접 코딩 ❌
- 복잡한 기술적 구현 ❌

**모델:** GLM-5

### 2.2 코딩 MCP

**책임:**
- PM의 기획서 기반 코드 구현
- 테스트 코드 작성
- 배포 스크립트 작성
- 기술적 문제 해결

**실행 방식:**
```javascript
sessions_spawn({
  runtime: "acp",  // ACP harness (Claude Code / Codex)
  agentId: "coder",  // 설정된 코딩 에이전트
  task: "PRD 내용...",
  mode: "run"  // 또는 "session" (지속적 대화)
})
```

**모델:** GLM-5 (MCP 설정에서 지정)

### 2.3 자동화 에이전트들 (VPS에서 실행)

**유형:**

| 유형 | 실행 방식 | 예시 |
|------|-----------|------|
| **Cron** | 주기적 실행 | 매일 리포트 생성 |
| **Daemon** | 지속 실행 | 웹훅 수신 대기 |
| **API** | 요청 시 실행 | REST API 서버 |
| **Watcher** | 이벤트 감지 | 파일 변경 시 처리 |

### 4.1 코어 시스템

### 4.2 자동화 에이전트 프레임워크

**Node.js 기반:**
```javascript
// 템플릿: daemon-agent.js
class AutomationAgent {
  constructor(config) {
    this.name = config.name;
    this.interval = config.interval;
  }

  async run() {
    // 코딩 MCP가 구현
  }

  async start() {
    setInterval(() => this.run(), this.interval);
  }
}
```

**Python 기반:**
```python
# 템플릿: daemon_agent.py
class AutomationAgent:
    def __init__(self, config):
        self.name = config['name']
        self.interval = config['interval']

    async def run(self):
        # 코딩 MCP가 구현
        pass

    async def start(self):
        while True:
            await self.run()
            await asyncio.sleep(self.interval)
```

## 9. 예시: 트위터 멘션 모니터링 자동화

### 9.1 PRD (PM 작성)

```markdown
# 트위터 멘션 모니터링 자동화

## 요구사항
- 특정 키워드 멘션 감지
- 텔레그램으로 실시간 알림
- 로그 저장

## 기술 스택
- Node.js
- Twitter API v2
- Telegram Bot API

## 실행 방식
- Daemon (지속 실행)
- 5분마다 폴링

## 알림 형식
"[트위터] @user님이 '키워드' 언급:
내용 미리보기..."
```

### 9.2 코딩 MCP 작업

```javascript
sessions_spawn({
  runtime: "acp",
  agentId: "coder",
  task: `
    PRD에 따라 트위터 멘션 모니터링 자동화 구현

    경로: /root/.openclaw/workspace/automation/agents/twitter-monitor/

    필수 파일:
    - index.js (메인 로직)
    - config.json (설정)
    - package.json

    기능:
    1. Twitter API v2로 멘션 검색
    2. 새 멘션 감지 시 Telegram API 호출
    3. 로그 파일에 기록

    에러 처리:
    - API 제한 시 대기
    - 네트워크 오류 시 재시도
  `
})
```

### 9.3 배포

```bash
# PM이 실행
cd /root/.openclaw/workspace/automation/agents/twitter-monitor
npm install
pm2 start index.js --name twitter-monitor
pm2 save
```

## 하지 않는 것

- ❌ 직접 코딩
- ❌ 복잡한 기술적 구현
- ❌ 세부적인 기술 결정 (코딩 MCP에 위임)
